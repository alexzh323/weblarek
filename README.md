Структура проекта:
- src/ — исходные файлы проекта
- src/components/ — папка с JS компонентами

# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные
В ходе анализа проекта было установлено, что в приложении используются две основные сущности, описывающие данные: товар и покупатель.

#### Интерфейс товара (IProduct)
Интерфейс содержит все данные о товаре. Необходим для описания данных о товаре, полученных с сервера. 

```typescript
interface IProduct {
  id: string;          // уникальный номер товара
  description: string; // полное описание товара
  image: string;       // путь к изображению товара
  title: string;       // название товара
  category: string;    // категория товара
  price: number | null;// цена товара, для бесценных товаров равна null
}
```

#### Интерфейс покупателя (IBuyer)
Интерфейс содержит все данные о покупателе. Задает структуру для данных покупателя, собираемых из форм оформления заказа.

```typescript
type TPayment = 'card' | 'cash' | '';

interface IBuyer {
  payment: TPayment;   // тип платежа: наличные ('cash'), карта ('card') или пустая строка (''), если поле не заполнено
  email: string;       // электронная почта покупателя
  phone: string;       // телефонный номер покупателя
  address: string;     // адрес доставки покупателя
}
```

#### Классы для хранения данных
Для учёта данных в приложении используются три класса, которые будут разделены между собой по смыслу и зонам ответственности:
ProductData - хранение товаров, которые можно купить в приложении;
CartData - хранение товаров, которые пользователь выбрал для покупки;
OrderData - данные покупателя, которые тот должен указать при оформлении заказа.

##### Класс ProductData
Класс служит для хранения всех товаров, которые можно купить в приложении, а также текущего товара, выбранного на данный момент.

Конструктор:  
`constructor()` создает пустой массив для списка товаров.

Поля класса:
`protected _products: IProduct[]` - массив всех доступных товаров. Используется для отображения каталога на главной странице.
`protected _currentProduct: IProduct | null` - текущий товар, выбранный для детального просмотра в модальном окне.
Поля класса защищенные, с ними можно взаимодействовать только через методы.

Методы класса:
`setProducts(products: IProduct[]): void` - сохраняет массив товаров в поле `_products`. Полностью заменяет предыдущие элементы массива новыми данными.
`getProducts(): IProduct[]` — возвращает массив всех товаров из модели.
`getProduct(id: string): IProduct | null` - возвращает товар по его уникальному `id`. Если товар с таким идентификатором не найден, возвращает `null`.
`setCurrentProduct(product: IProduct): void` - сохраняет выбранный товар в поле `_currentProduct` для последующего отображения в модальном окне.
`getCurrentProduct(): IProduct | null` - возвращает товар, выбранный в данный момент для детального просмотра, или `null`, если ни один товар не выбран.

##### Класс CartData 
Класс служит для хранения всех товаров, добавленных в корзину. Необходим для всех манипуляций с корзиной: добавления/удаления товаров, очистки корзины, проверки наличия в корзине конкретного товара, а также получения общего количества товаров в корзине и их суммарной стоимости. 

Конструктор: 
`constructor()` - создает пустой массив для списка товаров в корзине. 

Поля класса:
`protected _cartProducts: IProduct[]` - массив всех товаров, добавленных в корзину. Используется для отображения списка товаров в корзине. Поле защищенное, с ним можно взаимодействовать только через методы. 

Методы класса: 
`getCartProducts(): IProduct[]` - возвращает массив всех товаров в корзине. `addProduct(product: IProduct): void` - добавляет товар, полученный в параметре, в массив товаров корзины. 
`removeProduct(product: IProduct): void` - удаляет товар, полученный в параметре, из массива товаров корзины. 
`clearCart(): void` - удаляет все товары из корзины, превращая `_cartProducts` в пустой массив. 
`getTotalPrice(): number` - вычисляет суммарную цену всех товаров в корзине и возвращает её. Если корзина пуста, возвращает `0`. 
`getTotalNumber(): number` - возвращает количество товаров в корзине, то есть длину массива `_cartProducts`. 
`checkCartProduct(id: string): boolean` - проверяет наличие товара в корзине по его `id`, полученному в параметр метода, и возвращает `true` или `false`.

##### Класс OrderData 
Класс служит для хранения всех данных покупателя при оформлении заказа. Необходим для сохранения, получения, удаления и валидации данных покупателя. 

Конструктор: 
`constructor()` - инициализирует поля класса начальными пустыми значениями (`''`), данные заполняются с помощью методов по мере заполнения формы. 

Поля класса: 
`protected _payment: TPayment` - тип платежа: наличные ('cash'), карта ('card') или пустая строка (''), если поле не заполнено. 
`protected _email: string` - электронная почта покупателя. 
`protected _phone: string` - телефонный номер покупателя. 
`protected _address: string` - адрес доставки покупателя. 
Поля класса защищенные, с ними можно взаимодействовать только через методы.

Методы класса:
`setField<K extends keyof IBuyer>(field: K, value: IBuyer[K]): void` - универсальный метод сохранения данных для любого поля покупателя. Принимает название поля и его новое значение. Благодаря дженерикам, тип значения строго синхронизируется с типом выбранного поля интерфейса `IBuyer`.
`getFields(): IBuyer` - возвращает объект со всеми текущими данными покупателя.
`clearFields(): void` - очищает все данные покупателя, возвращая поля к их начальным пустым значениям (`''`).
`getInvalidFields(): TOrderValidationResult` - проводит валидацию заполненных полей. Поле считается невалидным, если его значение является пустой строкой. Возвращает объект, где ключами являются названия невалидных полей, а значениями - текст ошибок. Если все поля заполнены верно, возвращает пустой объект. 
Вспомогательные типы данных: 
```typescript 
type TOrderValidationResult = {
payment?: string; // Текст ошибки для способа оплаты 
address?: string; // Текст ошибки для адреса доставки
email?: string; // Текст ошибки для электронной почты
phone?: string; // Текст ошибки для номера телефона
}
```

### Слой коммуникации
#### Класс LarekApi 
Класс использует методы класса Api для отправки специализированных запросов на сервер.

Конструктор:
`constructor(api: IApi)` - принимает экземпляр класса Api, соответствующий интерфейсу IApi, и сохраняет его в своем поле.

Поля класса:
`protected _api: IApi` - защищенное поле, необходимое для дальнейших сетевых запросов.

Методы класса:
`getProductList(): Promise<IProductListResponse>` - выполняет GET-запрос на эндпоинт /product/. Возвращает промис с объектом, содержащим массив товаров (`IProduct[]`) и их общее количество.
`orderProducts(order: IOrderRequest): Promise<IOrderResponse>` - выполняет POST-запрос на эндпоинт /order/. Передает в теле запроса данные о заказе и покупателе (на основе`IBuyer`), возвращает промис с ответом от сервера, подтверждающим успешную покупку.
Вспомогательные типы данных: 
```typescript 
type IProductListResponse = {
  total: number;
  items: IProduct[]; 
}
type IOrderRequest = IBuyer & {
  total: number;
  items: string[]; 
}
type IOrderResponse ={
  id: string;
  total: number;
}
```
